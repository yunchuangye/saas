# GuJia.App 下一步开发建议

> 最后更新：2026-03-19
>
> 本文档基于现有代码分析，按优先级提出后续开发目标，分为**紧急修复**、**核心流程完善**、**功能扩展**、**体验优化**四个层次。

---

## 一、紧急修复（P0 — 影响核心业务流程）

### 1.1 个人客户选择中标评估公司

**问题**：申请详情页展示了收到的报价列表，但"选择此评估公司"按钮未实现。客户无法完成竞价闭环。

**建议方案**：
- 在申请详情页报价卡片上添加"选择中标"按钮
- 调用 `bids.award` API（已实现），传入 `bidId`
- 中标后自动将项目状态更新为 `awarded`，并向评估公司发送通知

**涉及文件**：`frontend/app/dashboard/customer/applications/[id]/page.tsx`

---

### 1.2 银行/投资机构需求发起页

**问题**：`bank/demand/new` 和 `investor/demand/new` 页面已有 UI 但未对接 API，机构用户无法通过平台发起评估需求。

**建议方案**：
- 复用个人客户"申请评估"的五级联动组件
- 对接 `projects.create` API，增加机构特有字段（贷款金额、项目用途等）
- 提交后跳转到项目列表页

**涉及文件**：`frontend/app/dashboard/bank/demand/new/page.tsx`、`frontend/app/dashboard/investor/demand/new/page.tsx`

---

### 1.3 个人/机构资料编辑

**问题**：所有角色的 `profile` 页面均为静态展示，`account` 页面未对接修改密码 API。

**建议方案**：
- 对接 `auth.updateProfile` API 实现资料编辑
- 对接 `auth.changePassword` API 实现密码修改
- 头像上传对接文件上传 API

---

## 二、核心流程完善（P1 — 影响用户体验的主要功能）

### 2.1 项目全流程状态流转

当前项目状态流转逻辑分散，建议梳理并完整实现以下流程：

```
客户提交申请（pending）
    ↓ 管理员/系统开启竞价
竞价中（bidding）
    ↓ 客户选择中标（bids.award）
已中标（awarded）
    ↓ 评估公司确认接单
现场勘察（surveying）
    ↓ 评估公司提交报告
报告编制（reporting）
    ↓ 机构/管理员审核
审核中（reviewing）
    ↓ 审核通过
已完成（completed）
```

**缺失环节**：
- 管理员开启竞价的操作入口
- 评估公司"确认接单"操作
- 勘察完成后上传勘察记录
- 客户进度跟踪页面（`customer/progress`）对接 API

---

### 2.2 报告详情页

**问题**：客户、银行、投资机构均有报告列表，但点击后无详情页，无法在线预览报告内容。

**建议方案**：
- 创建 `reports/[id]/page.tsx` 详情页（各角色共用）
- 展示报告基本信息、估价结果、附件文件
- 支持在线预览 PDF（使用 `react-pdf` 或 iframe）
- 支持下载报告文件

---

### 2.3 银行/投资机构首页对接 API

**问题**：`bank/page.tsx` 和 `investor/page.tsx` 均为静态页面，统计数据不真实。

**建议方案**：
- 对接 `dashboard.stats` API（已支持按角色返回不同数据）
- 对接 `dashboard.activityChart` API
- 对接 `dashboard.recentProjects` API

---

### 2.4 管理员首页对接 API

**问题**：`admin/page.tsx` 为静态展示，系统状态卡片均为硬编码。

**建议方案**：
- 对接 `dashboard.stats` API
- 添加实时系统状态监控（MySQL/Redis 连接状态、队列长度）
- 对接 `crawl.getDashboardStats` 展示爬虫运行状态

---

### 2.5 消息通知推送

**问题**：通知目前仅支持手动刷新，重要事件（新报价、中标、报告完成）无实时推送。

**建议方案**：
- 后端集成 **Server-Sent Events（SSE）** 或 **WebSocket**
- 关键事件触发时自动推送通知：
  - 新报价到达 → 通知客户
  - 中标 → 通知评估公司
  - 报告提交 → 通知审核方
  - 报告审核通过 → 通知客户

---

## 三、功能扩展（P2 — 提升平台价值的新功能）

### 3.1 销售推广模块前端实现

后端 `sales.ts` 已实现完整的销售推广 API，但前端均为静态页面，建议按优先级逐步实现：

| 功能 | 角色 | 优先级 |
|------|------|--------|
| 评估师微站（个人主页） | 评估公司 | 高 |
| 邀请码 + 分享海报 | 个人客户 | 高 |
| 贷款计算器 | 银行 | 中 |
| 营销活动管理 | 评估公司 | 中 |
| 投资路演 Pitchbook | 投资机构 | 低 |
| 拼团购买 | 个人客户 | 低 |

---

### 3.2 数据分析增强

**建议**：
- 银行/投资机构的"数据分析"页面（`analytics`）对接 `ai-features.getPriceTrend` 和 `property-search.*` API
- 增加区域价格热力图（基于 217 万条房屋数据）
- 增加楼盘价格走势对比图
- 增加评估公司业绩排行榜

---

### 3.3 App 下载页

**问题**：`/app-download` 页面已有路由，但内容未知，建议完善移动端下载引导页。

---

### 3.4 系统设置完善

**建议**：
- 对接平台基础配置（平台名称、Logo、联系方式）
- 对接邮件/短信通知配置
- 对接 OSS 存储配置（当前需手动修改 `.env`）
- 对接 OpenAI API Key 配置

---

### 3.5 忘记密码流程

**建议**：
- 实现手机短信验证码重置密码
- 或实现邮箱验证码重置密码
- 对接 `auth` 路由中的重置逻辑

---

## 四、体验优化（P3 — 提升产品质感）

### 4.1 表单验证增强

- 申请评估表单增加实时校验（手机号格式、面积范围等）
- 注册表单增加用户名重复检测（实时）
- 报价金额增加合理范围提示

### 4.2 加载状态优化

- 各列表页增加骨架屏（Skeleton）
- 操作按钮增加防重复提交（已提交后禁用）
- 页面切换增加过渡动画

### 4.3 移动端适配

- 当前 UI 以桌面端为主，建议对申请评估、报告查看等高频页面进行移动端响应式优化

### 4.4 搜索与筛选

- 项目列表增加按状态、时间、关键词筛选
- 报告列表增加按类型、时间筛选
- 用户列表增加搜索功能

### 4.5 操作确认与撤销

- 删除操作增加二次确认弹窗
- 提交申请后提供短时间内撤销功能
