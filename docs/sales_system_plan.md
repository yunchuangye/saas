# gujia.app 营销推广工具系统规划

## 1. 系统定位与目标

为 gujia.app 平台构建一套全方位的营销推广工具系统（Sales System），旨在帮助平台上的不同角色（个人、评估公司、银行、投资机构）进行业务拓展、品牌曝光、客户获取和转化。

## 2. 目录结构设计

### 前端目录 (`/frontend/app/dashboard/[role]/sales/`)
每个角色工作台下新增 `sales` 目录，作为该角色的营销工具主入口。

```
frontend/app/dashboard/
├── appraiser/sales/      # 评估公司营销工具
├── bank/sales/           # 银行机构营销工具
├── investor/sales/       # 投资机构营销工具
└── customer/sales/       # 个人客户营销工具
```

### 后端目录 (`/backend/server/routers/sales.ts`)
新增 `sales` 路由模块，处理所有营销相关的 API 请求。

## 3. 分角色功能规划

### 3.1 个人客户 (Customer)
**核心诉求**：分享赚佣金、邀请好友、获取评估优惠。
- **邀请返佣 (Referral Program)**：生成专属邀请海报/链接，邀请好友注册或下单获取佣金/积分。
- **拼团评估 (Group Valuation)**：发起小区批量评估拼团，达到人数享受折扣。
- **房产估值卡片分享**：将自己的房产估值结果生成精美卡片分享到社交媒体（隐藏敏感信息）。

### 3.2 评估公司 (Appraiser)
**核心诉求**：获客、品牌展示、专业能力背书。
- **微官网/微名片 (Micro-site)**：一键生成评估公司/评估师的专属展示页（含资质、案例、评价）。
- **营销海报生成器 (Poster Generator)**：提供行业报告、政策解读、估价趋势等模板，一键生成带公司二维码的海报。
- **客户CRM与线索追踪 (Lead Tracking)**：追踪通过分享链接访问的潜在客户，记录访问轨迹。
- **优惠券/体验券发放**：向潜在客户定向发放“免费初估券”或“评估折扣券”。

### 3.3 银行机构 (Bank)
**核心诉求**：房贷业务拓展、获客引流。
- **房贷计算器+估值插件 (Widget)**：生成可嵌入银行公众号/小程序的房产估值与房贷计算组件。
- **联名营销活动 (Co-marketing)**：与平台联合发起“查房价送权益”活动，获取高意向房贷客户线索。
- **区域市场报告生成**：生成带有银行 Logo 的区域房产市场分析报告，用于客户维护。

### 3.4 投资机构 (Investor)
**核心诉求**：资产处置、项目寻源、行业影响力。
- **资产推介册 (Asset Pitchbook)**：将不良资产或投资标的生成标准化的推介长图或 H5。
- **投资洞察简报 (Insight Newsletter)**：基于平台数据生成投资视角的市场简报，用于邮件营销。
- **项目合作邀请码**：生成特定项目的定向邀请链接，招募联合投资人或处置服务商。

## 4. 数据库设计扩展 (Schema)

需要在 `schema.ts` 中新增以下表（或在现有表中扩展）：
1. `sales_campaigns`：营销活动表
2. `sales_leads`：营销线索表（记录转化）
3. `sales_materials`：营销素材表（海报、文章等）
4. `user_invitations`：邀请关系表

## 5. 实施步骤

1. **后端开发**：创建 `sales.ts` 路由，定义基础的获取素材、生成链接、记录线索等接口。
2. **前端公共组件**：开发海报生成器、分享组件、数据统计图表等公共组件。
3. **分角色页面开发**：逐一实现四个角色的 `sales` 页面。
4. **导航集成**：修改 `roles.ts`，将“营销推广”加入各角色的侧边栏导航。
