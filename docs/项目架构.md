## 三、认证机制

- 使用 **JWT（JSON Web Token）** 进行无状态认证
- Token 有效期 7 天，存储在客户端 Cookie 中
- 后端 tRPC 中间件验证 Token，注入 `ctx.user`
- 权限分三级：`publicProcedure`（无需登录）、`protectedProcedure`（需登录）、`adminProcedure`（需管理员角色）

---

## 四、数据流

```
用户操作
  → 前端 tRPC Client（类型安全调用）
  → HTTP POST /trpc/{router}.{procedure}
  → 后端 tRPC Router（鉴权中间件）
  → Drizzle ORM 查询
  → MySQL 数据库
  → 返回类型安全结果
  → React Query 缓存更新
  → 前端 UI 自动刷新
```

## 五、环境变量说明

### backend/.env

| 变量 | 说明 | 默认值 |

| `DB_HOST` | MySQL 主机 | 127.0.0.1 |

## 六、本地开发快速启动

```bash
# 首次初始化（安装依赖 + 启动服务）
./setup.sh init

# 日常启动
./setup.sh start

# 查看服务状态
./setup.sh status

# 提交代码到 GitHub
./setup.sh sync "feat: 新功能描述"

# 查看后端日志
./setup.sh logs backend

# 进入数据库
./setup.sh db
```
